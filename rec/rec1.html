Función sumar recursiva

Se desea implementar una función recursiva que sume todos los elementos de un arreglo de enteros: 

static int sumar(int arr[]);

Como referencia, esta sería la implementación iterativa: 

// Versión iterativa de sumar().
static int sumarIter(int arr[]) {
    int suma = 0;

    for (int i=0; i < arr.length; i++)
        suma = suma + arr[i];

    return suma;
}

Caso base y caso recursivo

Para pensar la función recursiva, hay que determinar el caso base y el caso recursivo.

La pregunta para descubrir el caso base es:

Caso base: ¿cuál es el tamaño mínimo del problema (esto es, el tamaño mínimo del arreglo) para el que se puede dar una solución inmediata?

Y la pregunta para descubrir el caso recursivo:

Caso recursivo: ¿si la función ya supiera sumar arreglos de longitud \(N\), qué debería hacer para sumar un arreglo de longitud \(N+1\)?

Las respuestas para el caso de la suma son:

  • Caso base: se puede dar una solución inmediata si el arreglo es de tamaño 1, ya que: la suma de un arreglo con un solo elemento es, directamente, ese elemento, arr[0]: 

    suma([7]) = 7
suma([3]) = 3
suma([0]) = 0
suma([-1]) = -1

  • Caso recursivo: si por ejemplo sumar un arreglo de tres elementos (\(N = 3\)) es ya un caso resuelto, se puede dar una solución a un arreglo de \(N + 1 = 4\) elementos sumando al primer elemento la suma de los tres restantes: 

    suma([7, 3, 9, 5])  =  7 + suma([3, 9, 5])

Recursividad con arreglo por copia

El código del caso base es trivial: 

// Caso base solamente.
static int sumarRec1(int arr[]) {
    if (arr.length == 1) {
        return arr[0];
    }
}

Para el caso recursivo se debe saber cómo “achicar” el problema. Según lo explicado, en la suma se achica sumando “el resto del arreglo menos el primer elemento”: 

// Pseudo-código del caso recursivo
if arreglo.length > 1:
    return arreglo[0] + sumarRec1(“mismo arreglo sin el primer elemento”);

O, usando el anterior arreglo de ejemplo: 

// Pseudo-código con un arreglo de ejemplo
arreglo = [7, 3, 9, 5];
if arreglo.length > 1:
    return arreglo[0] + sumarRec1([3, 9, 5]);

En Java, para pasar de [7, 3, 9, 5] a [3, 9, 5] la única opción es realizar una copia del arreglo, pues no se le pueden eliminar elementos.

Se podría implementar la siguiente función auxiliar: 

// Devuelve una copia del arreglo sin su primer elemento.
static int[] borrarPrimero(int[] arreglo) {
    return null;
}

Con esta función, el pseudo-código quedaría tal que: 

// Pseudo-código usando borrarPrimero().
arreglo = [7, 3, 9, 5];
if arreglo.length > 1:
    return arreglo[0] + sumarRec1(borrarPrimero(arreglo));

sumarRec1

La versión final de sumarRec1() sería: 

static int sumarRec1(int arr[]) {
    if (arr.length == 1) {
        return arr[0];
    }
    if (arr.length > 1) {
      return arr[0] + sumarRec1(borrarPrimero(arr));
    }
    return 0;  // En caso de que nos pasen un arreglo vacío.
}

sumarRec2

Notar el cambio de prototipo y función auxiliar: 

static int sumarRec2(int arr[]) {
    if (arr.length == 0) {
        return 0;
    }
    return sumarAux(arr, 0);
}

private static int sumarAux(int arr[], int pos) {
    if (pos == arr.length - 1) {
        return arr[pos];
    }
    return arr[pos] + sumarAux(arr, pos + 1);
}

private static int minimoAux1(int arr[], int pos) {
    if (pos == arr.length - 1) {
      return arr[pos];
    }
    return Math.min(arr[pos], minimoAux1(arr, pos + 1));

minimoRec2

static int minimoRec2(int arr[]) {
    if (arr.length == 0) {
        return -1;  // ¿Qué valor usar?
    }
    return minimoAux2(arr, 1, arr[0]);
}

private static int minimoAux2(int arr[], int pos, int minimoParcial) {
    int nuevoParcial = Math.min(arr[pos], minimoParcial);
    if (pos == arr.length - 1) {
      return nuevoParcial;  // Es el final.
    }
    return minimoAux2(arr, pos + 1, nuevoParcial);
}
